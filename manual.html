Basic setup

  1. Install Forge (requires Ruby and RubyGems): 
    $ gem install forge
  2. Create your new theme project: 
    $ forge create themename
  3. Link to your WordPress theme folder: 
    $ forge link /path/to/wordpress/wp-content/themes/my-theme
  4. Watch for changes and start developing! 
    $ cd themename
    $ forge watch

How Forge works

All of your development work takes place in the source directory of your project. There are four folders you'll work from, each of which has a special function:

  • templates: Any and all WordPress templates for your theme
  • assets: Images, JavaScript/CoffeeScript, and CSS/Sass
  • functions and includes: Your functions.php file, along with any libraries or external code you need for your theme

Templates

The templates folder serves as a way to organize your template files. Any file you place in this directory will end up at the root of your theme. So that means that 

source/templates/index.php
source/templates/pages/my_template.php
source/templates/this/is/getting/ridiculous.php
will be translated to 
your_theme/index.php
your_theme/my_template.php
your_theme/ridiculous.php

Forge gives you some default folders that we think make sense - but feel free to organize however you'd prefer!

Assets

Assets are your JavaScript, CSS, and image files. Forge allows you to use CoffeeScript, Sass, and Sprockets on your theme painlessly. If you want to stick with plain CSS and JavaScript, you can do that as well.

Assets are split into three main categories:

  • Stylesheets
  • JavaScript
  • Images

Stylesheets

Stylesheets in Forge use Sass, a language that makes stylesheet development quicker and easier. In addition, Forge supports Compass and Blueprint mixins to speed things up even more.

Note: Forge supports both the Sass (.sass) and SCSS (.scss) syntax. Just change the filename to whatever.sass, and you're all set!

With Forge, the only stylesheet that will be included with your theme is the main stylesheet (style.css), which is located at 

source/assets/stylesheets/style.css.scss

Sass supports importing of partial stylesheets. Forge allows as many partials you want for breaking up your CSS into an organized structure. For example, assume the following files exist: 

source/assets/stylesheets/_pages.scss
source/assets/stylesheets/_reset.scss
source/assets/stylesheets/style.css.scss

Partial stylesheets are typically indicated with a leading underscore. Including the partials in your main stylesheet is simple: 

(in source/assets/stylesheets/style.css.scss)
@import "reset";
@import "pages";

body { color: #333; }
...

style.css.scss will be processed, a Theme header will be added based on the values in config.json, and the result output to 

your_theme/style.css

Read more about Sass, Compass and Blueprint

JavaScript

JavaScript in Forge harnesses the power of CoffeeScript and Sprockets. The bulk of your work will be done in 

source/assets/javascripts/theme.js

In theme.js, you can include any other JavaScript or CoffeeScript files you'll need in your final theme. For example, with these files: 

source/assets/javascripts/slider.js
source/assets/javascripts/tags.coffee
source/assets/javascripts/theme.js

Including other files is as easy as

(in source/assets/javascripts/theme.js)
//= require 'slider'
//= require 'tags'

$('#more .javascript .here').show();
...

theme.js will be processed and output to

your_theme/javascripts/theme.js

Forge also processes an admin JavaScript file, in case you need a separate file for your Theme Options:

source/assets/javascripts/admin.js

This is processed the same way as theme.js, but is output to

your_theme/javascripts/admin.js

Read more about CoffeeScript and Sprockets.

Images

Images in Forge are not too tricky - everything in

source/assets/images/

will be copied, unchanged to

your_theme/images/

Note: The only exception to this rule is your screenshot file - named screenshot.png, screenshot.jpg, screenshot.jpeg, or screenshot.gif. This file will be copied directly to the root of your theme.

To clarify with an example. If you had the following setup:

source/assets/images/photo.jpg
source/assets/images/screenshot.png
source/assets/images/animals/wild/lion.jpg
source/assets/images/food/hamburger.png

It would translate to the following:

your_theme/images/photo.jpg
your_theme/images/animals/wild/lion.jpg
your_theme/images/food/hamburger.png
your_theme/screenshot.png

Functions

Functions in Forge are pretty simple:

source/functions/functions.php

becomes

your_theme/functions.php

We plan on doing more here later, so stay tuned!

Includes

Again, pretty simple. Everything here is directly copied from 

source/includes/

into

your_theme/includes/

So, a quick example:

source/includes/struts/options.php
source/includes/theme_library.php

becomes

your_theme/includes/struts/options.php
your_theme/includes/theme_library.php
