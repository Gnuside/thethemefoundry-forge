Basic setup

  1. Install Forge (requires Ruby and RubyGems): 
    $ gem install forge
  2. Create your new theme project: 
    $ forge create themename
  3. Link to your WordPress theme folder: 
    $ forge link /path/to/wordpress/wp-content/themes/my-theme
  4. Watch for changes and start developing! 
    $ cd themename
    $ forge watch

How Forge Works

All of your development work goes on in the source directory of your project. There are 4 different folders you'll work with in your Forge project, each of which has a special function:

  • templates: Any and all WordPress templates for your theme
  • assets: Images, JavaScript/CoffeeScript, and CSS/SASS
  • functions: Your functions.php file
  • includes: Any libraries or external code you need for your theme

Let's take a look at what each section does.

Templates

The templates folder serves as a way to organize your template files. Any file you place in this directory will end up at the root of your theme. So that means that 

source/templates/index.php
and 
source/templates/pages/my_template.php
and even 
source/templates/this/is/getting/ridiculous.php
will be translated to 
your_theme/index.php
your_theme/my_template.php
your_theme/ridiculous.php

Forge gives you some default folders that we think make sense - but feel free to organize until your OCD little heart is content!

Assets

The assets folder allows you to use CoffeeScript, SASS, and Sprockets on your theme painlessly. Of course, you don't have to use anything beyond JavaScript and plain old CSS - it's up to you!

Stylesheets

Forge only makes use of one stylesheet: assets/stylesheets/style.css.scss, which is processed by SASS and compiled to your style.css file. That means you get the wonderfulness of SASS's partial stylesheets, mixins - all of it! Also included are Compass and Blueprint standard libraries for mixing in.

In addition, edit the settings in config.json (at the root of your Forge project) and they'll be reflected in your style.css header

JavaScripts

For theme JavaScript, everything not related to the admin panel goes in assets/javascripts/theme.js, which is processed by Sprockets and compiled to 

your_theme/javascripts/theme.js
You can harness the power of Sprockets includes and requires to generate a single JavaScript file for your theme

In addition, assets/javascripts/admin.js will compile to 

your_theme/javascripts/admin.js
in case you need JavaScript for your theme options

Images

Images aren't too tricky - everything in your assets/images/ folder will be compiled to 

your_theme/images/
exactly how you have it in your source folder. The only exception is screenshot.png/jpg/jpeg/gif. That will be automagically copied to 
your_theme/screenshot.png

Functions

Functions are dead simple. Your functions/functions.php file is copied to the root of your theme. We plan on doing more here later, so stay tuned :).

Includes

Again, pretty simple. Everything here is directly copied as-is to 

your_theme/includes/

That's it! That's all there is to Forge! Happy theming!
